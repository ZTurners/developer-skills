# Python Script Template

Used by `compile-skill` Step 4 as the structural blueprint for every generated script
under `compiled/scripts/`.
Replace all `<...>` placeholders with content derived from the source SKILL.md.

---

## File Header

```python
"""
<script-name>.py — Deterministic implementation compiled from <skill-name>/SKILL.md.

Generated by the compile-skill skill.
Source skill : <relative path to SKILL.md>
Compiled on  : <ISO-8601 date — use a static placeholder if date is non-deterministic>

Behavioral contract
-------------------
This script implements exactly the deterministic steps from the source skill.
Same inputs always produce the same outputs. No random, time-based, or
environment-dependent values appear in any output-producing code path.

Usage
-----
    python compiled/scripts/<script-name>.py --input <path-to-input-file>
    cat input.txt | python compiled/scripts/<script-name>.py
"""
```

## Module Structure (in order)

```python
from __future__ import annotations

import argparse
import json
import sys
from typing import TypedDict  # or dataclasses if preferred


# ---------------------------------------------------------------------------
# Constants — ALL output text lives here. Never hard-code strings in functions.
# ---------------------------------------------------------------------------

# Headings
HEADING_<SECTION>   = "## <exact heading text from skill>"

# Labels / column headers
LABEL_FILE          = "File"
LABEL_LINE          = "Line"
LABEL_RULE          = "Rule"
LABEL_SEVERITY      = "Severity"

# Error / fallback messages (verbatim from source skill)
ERROR_<NAME>        = "<exact error text from source skill>"
WARN_<NAME>         = "<exact warning text from source skill>"

# Separator / structural strings
SEP_TABLE_ROW       = "| {file} | {line} | {rule} | {severity} |"
SEP_TABLE_HEADER    = f"| {LABEL_FILE} | {LABEL_LINE} | {LABEL_RULE} | {LABEL_SEVERITY} |"
SEP_TABLE_DIVIDER   = "| --- | --- | --- | --- |"


# ---------------------------------------------------------------------------
# Type aliases (keep simple — plain dicts or TypedDicts)
# ---------------------------------------------------------------------------

class Finding(TypedDict):
    file:     str
    line:     int
    rule:     str
    severity: str   # "HIGH" | "MEDIUM" | "LOW"
    message:  str


# ---------------------------------------------------------------------------
# parse_* — file discovery, content extraction, pattern matching
# ---------------------------------------------------------------------------

def parse_input_files(workspace_root: str, pattern: str) -> list[str]:
    """Return sorted list of file paths matching `pattern` under `workspace_root`."""
    ...


def parse_<content_type>(raw: str) -> list[dict]:
    """Extract <content_type> entries from raw text. Returns list of dicts."""
    ...


# ---------------------------------------------------------------------------
# validate_* — rule checks; return findings, never raise on rule violations
# ---------------------------------------------------------------------------

def validate_<rule_name>(file_path: str, content: str, rules: list[str]) -> list[Finding]:
    """
    Apply <rule_name> to `content`. Return a (possibly empty) list of Findings.
    Only raises ValueError for programmer errors (bad arguments), never for
    ordinary rule violations.
    """
    ...


# ---------------------------------------------------------------------------
# select_* — ranking, filtering, sorting, deduplication
# ---------------------------------------------------------------------------

def select_by_severity(findings: list[Finding]) -> list[Finding]:
    """Return findings ordered HIGH → MEDIUM → LOW (stable sort)."""
    severity_rank = {"HIGH": 0, "MEDIUM": 1, "LOW": 2}
    return sorted(findings, key=lambda f: severity_rank.get(f["severity"], 99))


def select_<criterion>(items: list[dict], **kwargs) -> list[dict]:
    """Filter or rank `items` according to <criterion>."""
    ...


# ---------------------------------------------------------------------------
# format_* — output assembly; input is plain data, output is str
# ---------------------------------------------------------------------------

def format_table_header() -> str:
    """Return the Markdown table header for the findings table."""
    return f"{SEP_TABLE_HEADER}\n{SEP_TABLE_DIVIDER}"


def format_finding_row(finding: Finding) -> str:
    """Format a single finding as a Markdown table row."""
    return SEP_TABLE_ROW.format(
        file=finding["file"],
        line=finding["line"],
        rule=finding["rule"],
        severity=finding["severity"],
    )


def format_<output_section>(data: list[Finding]) -> str:
    """
    Assemble the full <output_section> string.
    Uses only constants and format_* helpers — no inline strings.
    """
    if not data:
        return ERROR_<NAME>
    lines = [HEADING_<SECTION>, format_table_header()]
    lines.extend(format_finding_row(f) for f in data)
    return "\n".join(lines)


def format_report(sections: dict[str, str]) -> str:
    """Concatenate all formatted sections in the order defined by the source skill."""
    ordered_keys = [
        # List section keys in the exact order the source skill specifies output
        "<section-1>",
        "<section-2>",
    ]
    return "\n\n".join(sections[k] for k in ordered_keys if k in sections)


# ---------------------------------------------------------------------------
# main — CLI entry point; the only function that performs I/O
# ---------------------------------------------------------------------------

def main() -> None:
    parser = argparse.ArgumentParser(
        description="<one-line description from source skill>"
    )
    parser.add_argument(
        "--input",
        metavar="FILE",
        default="-",
        help="Input file path (default: stdin)",
    )
    args = parser.parse_args()

    if args.input == "-":
        raw = sys.stdin.read()
    else:
        with open(args.input, encoding="utf-8") as fh:
            raw = fh.read()

    # 1. Parse
    items = parse_<content_type>(raw)

    # 2. Validate
    all_findings: list[Finding] = []
    for item in items:
        all_findings.extend(validate_<rule_name>(item["file"], item["content"], item["rules"]))

    # 3. Select / rank
    ranked = select_by_severity(all_findings)

    # 4. Format
    output = format_<output_section>(ranked)

    sys.stdout.write(output + "\n")


if __name__ == "__main__":
    main()
```

## Rules Checklist

Before finalizing the generated script, verify every item:

| # | Rule | Check |
|---|------|-------|
| 1 | All output text is a module-level constant | ☐ |
| 2 | Function order: `parse_*` → `validate_*` → `select_*` → `format_*` → `main` | ☐ |
| 3 | Every function is ≤ 30 lines | ☐ |
| 4 | No `random`, `datetime.now()`, `uuid`, or unordered iteration in output paths | ☐ |
| 5 | Functions take and return only `str`, `list`, or `dict` | ☐ |
| 6 | `main()` is the only function with I/O | ☐ |
| 7 | All error messages are verbatim from the source skill, stored as constants | ☐ |
| 8 | No `except: pass` or silent fallbacks | ☐ |
| 9 | `format_report()` respects the output section order from the source skill | ☐ |
| 10 | Script runs with `python compiled/scripts/<script>.py --input <file>` and writes to stdout | ☐ |

## Naming Derivation

| Source element | Generated name |
|----------------|----------------|
| Fixed heading `"## Findings"` | `HEADING_FINDINGS = "## Findings"` |
| Error `"No files found."` | `ERROR_NO_FILES = "No files found."` |
| Parse step: extract `.cpp` files | `parse_cpp_files(root: str) -> list[str]` |
| Validate step: check naming convention | `validate_naming_convention(...)` |
| Select step: sort by severity | `select_by_severity(findings)` |
| Format step: produce findings table | `format_findings_table(findings)` |
